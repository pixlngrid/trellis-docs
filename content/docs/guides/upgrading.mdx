Starting with v1.6.2, {vars.productName} includes a CLI upgrade command that updates framework files and dependencies in an existing project. For older versions, follow the manual steps below.

## Upgrade with the CLI

Run from your project root:

```bash title="Terminal"
npx create-trellis-docs@latest upgrade
```

This command:
- Copies updated framework files (components, lib, scripts, app routes)
- Merges new dependencies and scripts into your `package.json`
- Skips your user files (`config/`, `content/`, `public/`, `components/brand/`)

Preview what changes without modifying anything:

```bash title="Terminal"
npx create-trellis-docs@latest upgrade --dry-run
```

After upgrading, review the diff, install, and build:

```bash title="Terminal"
git diff
npm install
npm run build
```

:::tip
The CLI upgrade does not modify your `config/site.ts`. Check the version-specific sections below for any new config options you should add manually.
:::

## Manual upgrade steps

If you prefer to upgrade manually, or are upgrading from a version older than v1.6.2:

1. **Check the release notes** — Read the [release notes](/release-notes/) for all versions between your current version and the target.
2. **Follow the version-specific steps below** — Each section lists the exact files and dependencies to update.
3. **Install and build** — Run `npm install` then `npm run build` to verify everything works.

:::tip
To see the full latest template, scaffold a fresh project in a temporary directory with `npx create-trellis-docs@latest temp-docs` and diff it against your project. This is a reference copy for comparison — don't run the scaffolder inside your existing project.
:::

## Troubleshooting

### `lightningcss` or `@tailwindcss/oxide` errors on Windows

If your `.npmrc` contains `os = "linux"`, npm skips Windows-specific optional dependencies. Fix by running:

```bash title="Terminal"
npm install --os=win32
```

Or add the Windows binaries to your `package.json`:

```json title="package.json"
{
  "optionalDependencies": {
    "@tailwindcss/oxide-win32-x64-msvc": "^4.2.0",
    "lightningcss-win32-x64-msvc": "^1.31.1"
  }
}
```

### `.next/trace` EPERM error during build

On Windows, a stale Node.js process might hold a lock on the `.next` directory. Kill all Node processes and clean the build cache:

```bash title="Terminal"
taskkill /f /im node.exe
rm -rf .next
npm run build
```

### `vars is not defined` in MDX

Ensure the `MDXRemote` options include the `scope` property:

```tsx title="MDXRemote usage"
<MDXRemote
  source={content}
  components={mdxComponents}
  options={{
    scope: { vars: docVariables },
    mdxOptions: { ... },
  }}
/>
```

### Build fails with "Could not parse expression with acorn"

This error occurs when `rehype-mdx-code-props` is installed. It tries to parse code block meta strings like `{2,4-5}` as JavaScript expressions. Remove it and use the custom `rehypeCodeMeta` plugin from `lib/rehype-code-meta.ts` instead.
