Trellis includes several reusable React components you can import and use in your MDX files.

## Included Components

| Component | Location | Description |
|-----------|----------|-------------|
| [Glossary](/components/glossary/) | `components/custom/glossary.tsx` | Renders a searchable, alphabetically sorted glossary from a JSON data file |
| [FaqTableOfContents](/plugins/faq-index/) | `components/custom/faq-table-of-contents.tsx` | Renders a searchable FAQ question list from the FAQ index |
| [Feedback](/components/feedback/) | `components/custom/feedback.tsx` | Embeddable feedback widget for collecting page-level feedback |
| FlippingCard | `components/custom/flipping-card.tsx` | Interactive flip cards with quiz/option selection support |
| Search | `components/docs/search/search-dialog.tsx` | Search UI with dialog, powered by Fuse.js (Cmd+K) |

## Using Components in MDX

Import any component at the top of your MDX file:

```mdx
---
title: My Page
---

import { Glossary } from '@/components/custom/glossary';

## Glossary

<Glossary />
```

The `@/` alias resolves to your project root directory.

## Search

The search component is already wired into the navbar. It renders a search icon that opens a dialog with instant search results. You typically don't need to import this directly — it's built into the navbar and accessible via **Cmd+K** (Mac) or **Ctrl+K** (Windows/Linux).

## FlippingCard

Interactive cards that flip to reveal content on the back. Supports option selection (radio buttons) for quiz-style interactions. The component uses CSS grid layout and native HTML form elements.

```mdx
import { FlippingCard } from '@/components/custom/flipping-card';

<FlippingCard
  cards={[
    {
      front: { title: 'Question 1', content: 'What is Trellis?' },
      back: { title: 'Answer', content: 'A docs framework built on Next.js.' },
    },
  ]}
/>
```
