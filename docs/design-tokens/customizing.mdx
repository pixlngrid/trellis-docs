This guide walks through customizing design tokens to match your brand.

## Changing Brand Colors

Open `design-tokens.json` and update the `brand` section. The default uses a violet scale:

```json title="design-tokens.json"
{
  "brand": {
    "primary-50":  { "value": "#f5f3ff", "type": "color" },
    "primary-100": { "value": "#ede9fe", "type": "color" },
    "primary-200": { "value": "#ddd6fe", "type": "color" },
    "primary-300": { "value": "#c4b5fd", "type": "color" },
    "primary-400": { "value": "#a78bfa", "type": "color" },
    "primary-500": { "value": "#8b5cf6", "type": "color" },
    "primary-600": { "value": "#7c3aed", "type": "color" },
    "primary-700": { "value": "#6d28d9", "type": "color" },
    "primary-800": { "value": "#5b21b6", "type": "color" },
    "primary-900": { "value": "#4c1d95", "type": "color" }
  }
}
```

Replace the hex values with your brand's color scale. Use a tool like [Tailwind CSS Colors](https://tailwindcss.com/docs/colors) or [Coolors](https://coolors.co/) to generate a consistent scale.

After editing, regenerate the CSS:

```bash
yarn build-tokens
```

Or simply restart the dev server — it runs `build-tokens` automatically:

```bash
yarn start
```

## Changing Accent Colors

The `accent` section controls secondary colors used for interactive elements like tabs and links:

```json
{
  "accent": {
    "teal-200": { "value": "#99f6e4", "type": "color" },
    "teal-300": { "value": "#5eead4", "type": "color" },
    "teal-400": { "value": "#2dd4bf", "type": "color" },
    "teal-500": { "value": "#14b8a6", "type": "color" },
    "teal-600": { "value": "#0d9488", "type": "color" }
  }
}
```

:::tip
When choosing accent colors, ensure sufficient contrast against both light and dark backgrounds. The `400` shade is used most often.
:::

## Adding New Tokens

Add new categories or tokens by extending the JSON structure:

```json title="Adding a secondary brand color"
{
  "brand": {
    "primary-600": { "value": "#7c3aed", "type": "color" },
    "secondary-600": { "value": "#2563eb", "type": "color" }
  }
}
```

This generates:

```css
--brand-secondary-600-value: #2563eb;
```

You can then use it in `custom.css` or any component:

```css
.my-element {
  color: var(--brand-secondary-600-value);
}
```

## Changing Typography

Update the font family in the `typography` section:

```json
{
  "typography": {
    "fontFamily": {
      "name": "font-family",
      "value": "'Inter', 'Segoe UI', system-ui, sans-serif"
    }
  }
}
```

If using a web font, add the import in `src/css/custom.css`:

```css title="src/css/custom.css"
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap');
```

## Full Rebranding Checklist

1. **Colors** — Update `brand` and `accent` sections in `design-tokens.json`
2. **Typography** — Update `typography.fontFamily` (add web font import if needed)
3. **Logos** — Replace SVG files in `static/img/` (favicon, horizontal logo, dark variant)
4. **Config** — Update `title`, `tagline`, `url`, and `customFields.repoUrl` in `docusaurus.config.js`
5. **Regenerate** — Run `yarn build-tokens`
6. **Verify** — Run `yarn build` and check for errors, then `yarn start` for visual review

## Token Naming Conventions

Follow these conventions for consistency:

- **Colors**: `{category}-{shade}` → `primary-600`, `teal-400`
- **Spacing**: `spacing-{pixels}` → `spacing-16`, `spacing-24`
- **Border radius**: `border-radius-{pixels}` → `border-radius-8`
- **Border width**: `border-width-{pixels}` → `border-width-1`

The build script converts these to CSS variables by joining the full path with hyphens: `--brand-primary-600-value`, `--spacing-spacing-16-value`.
